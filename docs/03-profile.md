Личный профиль
У каждого авторизованного пользователя есть страница личного профиля. Редактировать профиль или просматривать его полную версию могут только сам владелец профиля и администраторы.

Рекрутеры не имеют доступа к редактированию профиля; они видят данные резюме только в режиме просмотра.


Профиль состоит из четырех разделов:

Me
Содержит обязательные встроенные атрибуты. Эти атрибуты присутствуют всегда, и их невозможно удалить. Примеры: имя, фамилия, местоположение, личное фото. Эти атрибуты должны быть реализованы на том же «движке» (например, их можно добавить в шаблон вакансии), но рекрутеры не могут их удалить.

Info
Содержит выбранные пользователем атрибуты из библиотеки атрибутов.
Кандидаты могут:
добавлять атрибуты в библиотеку или удалять их;
заполнять значения для этих атрибутов.
Например: балл по IELTS, навыки проведения презентаций, готовность к удаленной работе.

Projects
Пользователи могут вести список проектов.
Каждый проект включает:
Название.
Период (диапазон дат).
Описание (с поддержкой разметки Markdown).
Теги технологий (поддержка автодополнения для ранее введенных тегов; использование удобного UI-компонента для отображения тегов).
Проекты можно добавлять, редактировать и удалять.

CVs
Отображает все резюме, созданные кандидатом. На одну должность может приходиться не более одного резюме. Существующие резюме можно редактировать или удалять. Новые резюме можно создавать только для тех должностей, которые доступны кандидату. Каждая запись в списке служит ссылкой на соответствующую страницу резюме.


Auto-Save
Страницы личного профиля поддерживают функцию автосохранения:
изменения отслеживаются локально;
сохранение происходит каждые 5–10 секунд;
сохранение не должно выполняться при каждом нажатии клавиши;
механизм автосохранения должен использовать оптимистичную блокировку.
Система должна использовать оптимистичную блокировку. Каждая операция сохранения (атрибутов, позиций и автосохранения профилей):
передает текущий номер версии;
обновляет запись, если версия совпадает;
возвращает новый номер версии;
завершается с ошибкой, если версия изменилась.
Клиентское приложение должно корректно обрабатывать конфликты версий.
