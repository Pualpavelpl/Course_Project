Ключевая функция №1: Библиотека атрибутов
Библиотека атрибутов позволяет использовать одни и те же структурированные данные в профилях, описаниях должностей и резюме.

Все рекрутеры могут создавать, редактировать и удалять атрибуты.

Каждый атрибут включает:
Категорию (выбирается из предустановленного списка);
Название (уникальное для всей системы);
Описание;
Тип данных.

Примеры категорий: сертификация, знание предметной области, личная информация, гибкие навыки (soft skills). Поддерживаемые типы атрибутов:
Строка (простой текст в одну строку);
Текст (с разметкой Markdown);
Изображение (загрузка во внешнее облачное хранилище методом перетаскивания — drag-n-drop);
Число;
Дата;
Период (диапазон дат);
Логический тип (флажок/checkbox);
Выбор из списка (выпадающий список).

Поскольку библиотека может стать объемной, механизм выбора атрибутов должен поддерживать:
Поиск по начальным символам (префиксу);
Список недавно использованных атрибутов;
Фильтрацию по категориям.

Ключевая функция №2: Позиции
Позиции представляют собой настраиваемые шаблоны резюме.

Все рекрутеры работают с общим списком позиций. Рекрутер может:
создать новую позицию «с нуля» или скопировать существующую;
редактировать любую позицию;
удалить любую позицию.

Каждая позиция включает:
основную информацию, в том числе название и краткое описание;
правила доступа (позиция может быть общедоступной — для всех авторизованных пользователей — или иметь ограниченный доступ, регулируемый фильтрами);
атрибуты, выбранные из библиотеки атрибутов;
теги проектов (для отбора релевантных проектов), а также максимальное количество проектов, включаемых в сформированное резюме.
Примеры правил доступа:
числовое значение «Балл IELTS» > 7,0;
отмечен чекбокс «Удаленная работа»;
значение «Навыки проведения презентаций» в выпадающем списке — «Продвинутый уровень» (Advanced).

Доступные операторы фильтрации зависят от типа атрибута.

Для каждой вакансии отображается список созданных по ней резюме (доступен только рекрутерам и администраторам).

Кандидаты могут создавать резюме только для тех вакансий, к которым у них есть доступ. Если кандидат теряет доступ, созданные ранее резюме не удаляются, а скрываются в интерфейсе.

Было бы полезно добавить дополнительные атрибуты вакансий (например, «Компания», «Уровень» — Junior/Middle/Senior/C-level и т. д.) для фильтрации и сортировки вакансий в таблицах, однако это не является критически важным.

Ключевая функция №3: Генерация резюме
Резюме формируются автоматически на основе:
данных профиля кандидата (обязательные атрибуты, такие как имя);
выбранных атрибутов из библиотеки (значения либо подтягиваются автоматически из данных, указанных кандидатом, либо заполняются вручную);
проектов кандидата (с учетом фильтрации).

При редактировании атрибута в резюме он добавляется в профиль (если это необходимо; некоторые поля могут быть уже заполнены данными из профиля); атрибуты, отсутствующие в профиле, по умолчанию остаются пустыми.

Сгенерированное резюме должно:
иметь профессиональное оформление;
быть разделено на четко структурированные блоки;
содержать только релевантную информацию (атрибуты, соответствующие требованиям вакансии, и отфильтрованные проекты).
Когда кандидат открывает свое резюме:
Поля (атрибуты) автоматически заполняются данными из профиля.
Недостающую информацию можно ввести вручную непосредственно в поле;

Каждое поле в резюме можно редактировать на месте (в профиле хранится единое основное значение для этого атрибута). Если поле не заполнено, оно подсвечивается красным цветом.

Редактирование поля в резюме изменяет соответствующее значение в профиле.

Рекрутеры могут просматривать резюме только в режиме «только для чтения»; незаполненные поля подсвечиваются красным. Они не могут напрямую изменять резюме кандидатов (это доступно администраторам).

Необходимо предусмотреть отслеживание статусов резюме — например, через отдельное действие «Опубликовать», доступное только при условии, что заполнены все поля. Это действие делает резюме видимым для рекрутеров.d
